# Skills 디렉토리

## 스킬(Skill)이란?

스킬은 **반복적으로 사용되는 작업 패턴**을 문서화한 것입니다.

Claude가 특정 유형의 작업을 수행할 때 일관된 방식으로 처리하도록 돕습니다. CLAUDE.md에 모든 내용을 담으면 토큰이 낭비되므로, 필요할 때만 로드하는 **Lazy Loading** 방식으로 운영합니다.

---

## 스킬 문서를 만드는 시점

**세션 로그에서 동일한 패턴이 3회 이상 반복될 때** 스킬 문서로 분리합니다.

### 예시
1. 세션 로그에서 "API 엔드포인트 추가" 작업이 반복됨
2. 매번 같은 구조, 같은 체크리스트를 사용
3. → `skills/api-endpoint.md` 스킬 문서 생성

---

## 발전 경로

```
세션 로그 패턴      →    스킬 문서 분리      →    FRAMEWORK_RULES.md
(3회 이상 반복)          (프로젝트별 적용)         (모든 프로젝트 적용)
```

1. **세션 로그 패턴**: 작업 중 발견한 반복 패턴
2. **스킬 문서 분리**: 해당 프로젝트에서 재사용
3. **FRAMEWORK_RULES.md**: 모든 프로젝트에 적용할 만큼 범용적인 경우

---

## 사용법

### 1. 스킬 문서 생성

`skill.md.template`을 복사하여 새 스킬 문서를 만듭니다:

```bash
cp skill.md.template my-skill.md
```

### 2. CLAUDE.md에서 참조

CLAUDE.md의 Lazy Loading 섹션에 스킬을 등록합니다:

```markdown
### 상세 컨텍스트 참조 (필요 시 로드)
- API 엔드포인트 작업: `.ai/skills/api-endpoint.md`
- 테스트 작성: `.ai/skills/testing.md`
```

### 3. 세션에서 로드

Claude에게 스킬을 로드하도록 지시합니다:

```
"skills/api-endpoint.md를 읽고 시작해라"
```

---

## 스킬 문서 작성 가이드

좋은 스킬 문서는 다음을 포함합니다:

1. **명확한 적용 대상** — 언제 이 스킬을 사용하는가
2. **필수/권장/금지 규칙** — 해야 할 것과 하지 말아야 할 것
3. **코드 예시** — 좋은 예시와 나쁜 예시
4. **체크리스트** — 작업 완료 시 확인 사항

---

## 스킬 예시

| 스킬명 | 파일 | 적용 대상 |
|---|---|---|
| API 엔드포인트 | `api-endpoint.md` | REST API 추가/수정 |
| 테스트 작성 | `testing.md` | 단위/통합 테스트 |
| 에러 핸들링 | `error-handling.md` | 예외 처리 패턴 |
| 마이그레이션 | `migration.md` | DB 스키마 변경 |
